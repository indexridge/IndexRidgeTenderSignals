# Product Setup Checklist

## Phase 1 — public beta validation

- [x] Create local project.
- [x] Add risk/copyright policy.
- [x] Register official/open procurement source.
- [x] Build no-dependency official API fetcher.
- [x] Generate tender-alert sample CSV/JSON/Markdown.
- [x] Generate downloadable public-data pack sample files.
- [x] Create public beta pages with no checkout, no signup, no analytics, no outreach.
- [ ] Review sample usefulness and niche quality.
- [ ] Add further approved official/open procurement sources if needed.

## Phase 2 — monetisation prep, still no payment active

- [ ] Choose payment platform: Gumroad, Payhip, Stripe, or Lemon Squeezy.
- [ ] Draft checkout copy with mandatory disclaimer.
- [ ] Draft support/refund/cancellation wording.
- [ ] Decide first paid SKU: monthly tender alerts, one-off data packs, or both.
- [ ] User approves exact platform/copy before publication.

## Do-not-do list

- Do not copy tender attachments/PDFs/images/logos/maps/screenshots.
- Do not use proprietary competitor data.
- Do not imply official government affiliation.
- Do not claim complete coverage, guaranteed leads, guaranteed revenue, bid success, or compliance.
- Do not provide legal/procurement/bid-writing/financial/tax/compliance advice.
- Do not email prospects, add analytics/signup forms, run ads, or take payment without explicit approval.
